## Capability 1 — Skill Auto-Creation

**Goal:** turn a reusable multi-step session into a new SKILL.md.

### Trigger

Use when:
- the user explicitly asks to create/save a skill, or
- the session had 5+ tool calls across 3+ distinct phases and looks reusable

Skip if it was clearly one-off.

### Procedure

1. **Analyze the session**
   Extract:
   - user intent
   - ordered workflow
   - decision points
   - final outcome

2. **Identify the reusable pattern**
   Ask:
   - what varies between runs?
   - what stays constant?
   - is this repeatable outside this exact task?

3. **Draft the skill**
   Use this template:

   ```markdown
   ---
   name: <kebab-case-name>
   description: <one-line description with 3+ trigger phrases>
   ---

   ## When to Use This Skill
   <Trigger conditions>

   ## Procedure
   <Concrete numbered steps>

   ## Parameters
   <What varies>

   ## Learned From
   - Session: <session_id>
   - Date: <date>
   - Original request: "<original ask>"
   ```

4. **Propose**
   Ask:
   > "This looks like a reusable workflow. Save it as a skill?"

   Use `ask_user` with choices: `["Save skill", "Edit first", "Skip"]`.

5. **Save**
   Save to the user-level directory:
   ```bash
   mkdir -p ~/.copilot/skills/<name>
   ```
   Write:
   `~/.copilot/skills/<name>/SKILL.md`

### Quality Gates

- self-contained; no references like "in this session"
- at least 3 concrete procedural steps
- description includes diverse trigger phrases
- concise and actionable
